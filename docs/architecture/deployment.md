## Infrastructure Roles

| Service | Role | POC | Production |
|---------|------|-----|------------|
| **Vercel** | Frontend hosting + API routes | Yes (free tier) | Yes (Pro for 300s timeout) |
| **AWS S3** | Generated creative storage + uploaded asset storage | Yes (free tier: 5GB) | Yes |
| **OpenAI API** | DALL-E 3 image generation | Yes (~$0.50/run for 6 images) | Yes (or Adobe Firefly) |
| **Vercel Blob** | Alternative to S3 for simpler setup | Optional fallback | No (S3 preferred at scale) |
| **Redis (Upstash)** | Job queue, generation status cache | No (stateless POC) | Yes |
| **PostgreSQL** | Campaign history, user accounts, analytics | No | Yes (Vercel Postgres or PlanetScale) |
| **CDN** | Image delivery with edge caching | Vercel's built-in CDN | CloudFront in front of S3 |

## S3 Configuration

### Bucket Structure

```
adspark-{environment}/
в”ңв”Җв”Җ assets/                    # Uploaded input assets
в”Ӯ   в””в”Җв”Җ {campaign-id}/
в”Ӯ       в””в”Җв”Җ {filename}
в”ңв”Җв”Җ campaigns/                 # Generated output
в”Ӯ   в””в”Җв”Җ {campaign-id}/
в”Ӯ       в””в”Җв”Җ {product-slug}/
в”Ӯ           в””в”Җв”Җ {ratio}/
в”Ӯ               в”ңв”Җв”Җ creative.png
в”Ӯ               в””в”Җв”Җ thumbnail.webp
в””в”Җв”Җ manifests/                 # Generation manifests
    в””в”Җв”Җ {campaign-id}.json
```

### Access Pattern

| Operation | Method | Auth |
|-----------|--------|------|
| Upload assets (frontend вҶ’ S3) | Pre-signed PUT URL (generated by API route) | Temporary, scoped to key prefix |
| Serve creatives (S3 вҶ’ frontend) | Pre-signed GET URL (24hr TTL) | Read-only, time-limited |
| Pipeline write (API route вҶ’ S3) | IAM role / access key | Server-side only |

### Why Pre-Signed URLs?

The frontend never holds AWS credentials. The API route generates short-lived, scoped URLs:

```typescript
import { S3Client, PutObjectCommand, GetObjectCommand } from '@aws-sdk/client-s3';
import { getSignedUrl } from '@aws-sdk/s3-request-presigner';

// Upload: frontend gets a PUT URL, uploads directly to S3
const uploadUrl = await getSignedUrl(s3, new PutObjectCommand({
  Bucket: process.env.S3_BUCKET,
  Key: `assets/${campaignId}/${filename}`,
}), { expiresIn: 300 }); // 5 min

// Download: frontend gets a GET URL to display the image
const viewUrl = await getSignedUrl(s3, new GetObjectCommand({
  Bucket: process.env.S3_BUCKET,
  Key: `campaigns/${campaignId}/${productSlug}/${ratio}/creative.png`,
}), { expiresIn: 86400 }); // 24 hr
```

## Environment Variables

### Required

| Variable | Description | Example |
|----------|-------------|---------|
| `OPENAI_API_KEY` | OpenAI API key with DALL-E 3 access | `sk-proj-...` |
| `AWS_ACCESS_KEY_ID` | AWS IAM access key for S3 | `AKIA...` |
| `AWS_SECRET_ACCESS_KEY` | AWS IAM secret key | `wJal...` |
| `S3_BUCKET` | S3 bucket name | `adspark-production` |
| `S3_REGION` | AWS region | `us-east-1` |

### Optional

| Variable | Description | Default |
|----------|-------------|---------|
| `STORAGE_MODE` | `s3` or `local` | `local` |
| `LOCAL_OUTPUT_DIR` | Path for local storage mode | `./output` |
| `LOCAL_ASSETS_DIR` | Path for local asset lookup | `./assets` |
| `OPENAI_MAX_RETRIES` | Max retry attempts for DALL-E | `3` |
| `OPENAI_CONCURRENCY` | Max parallel DALL-E requests | `5` |
| `LOG_LEVEL` | Logging verbosity | `info` |
| `NEXT_PUBLIC_APP_URL` | Public URL (for CORS, SSE) | `http://localhost:3000` |

### Storage Mode Fallback

The pipeline uses a storage abstraction. When `STORAGE_MODE` is unset or `local`, it writes to the filesystem вҖ” no S3 credentials needed for local development:

```typescript
const storage = process.env.STORAGE_MODE === 's3'
  ? new S3Storage({ bucket: process.env.S3_BUCKET!, region: process.env.S3_REGION! })
  : new LocalStorage({ outputDir: process.env.LOCAL_OUTPUT_DIR || './output' });
```

## Local Development

### Prerequisites

```bash
node >= 20.x
npm >= 10.x
```

### Setup

```bash
git clone https://github.com/{user}/adspark.git
cd adspark
npm install
cp .env.example .env.local    # Add your OPENAI_API_KEY
npm run dev                    # http://localhost:3000
```

### Local Mode Behavior

- `STORAGE_MODE=local` (default): images saved to `./output/`, served via Next.js static file route
- No S3 credentials needed for development
- No Redis, no database, no external services beyond OpenAI
- Hot reload via Next.js dev server

### Minimum Viable Local Run

Only ONE env var is truly required for local dev:

```env
OPENAI_API_KEY=sk-proj-your-key-here
```

Everything else has sensible defaults. This means a reviewer can:

```bash
git clone && npm install && echo "OPENAI_API_KEY=..." > .env.local && npm run dev
```

Three commands to a running app.

## Cloud Deployment (Vercel)

### One-Click Deploy

```bash
# Install Vercel CLI
npm i -g vercel

# Deploy (prompts for env vars on first run)
vercel --prod
```

Or: connect the GitHub repo to Vercel dashboard for automatic deploys on push.

### Vercel Environment Variables

Set in Vercel dashboard вҶ’ Project вҶ’ Settings вҶ’ Environment Variables:

```
OPENAI_API_KEY     = sk-proj-...       (encrypted, server-only)
AWS_ACCESS_KEY_ID  = AKIA...           (encrypted, server-only)
AWS_SECRET_ACCESS_KEY = wJal...        (encrypted, server-only)
S3_BUCKET          = adspark-prod      (plain, server-only)
S3_REGION          = us-east-1         (plain, server-only)
STORAGE_MODE       = s3                (plain, server-only)
```

All secrets are encrypted at rest and only available to server-side code (API routes). The `NEXT_PUBLIC_` prefix is intentionally NOT used for any secret.

### Vercel Limits (Hobby Tier)

| Resource | Limit | Impact |
|----------|-------|--------|
| Function timeout | 60s | OK вҖ” parallel DALL-E calls complete in ~20s |
| Function memory | 1024 MB | OK вҖ” Sharp + Canvas fit comfortably |
| `/tmp` storage | 512 MB | OK вҖ” ephemeral image processing before S3 upload |
| Bandwidth | 100 GB/mo | OK вҖ” images served from S3/CDN, not Vercel |
| Deployments | Unlimited | OK |

## Secrets Handling

### Rules

1. **Never commit secrets.** `.env.local` is in `.gitignore`. `.env.example` contains only keys with empty values.
2. **Server-side only.** No `NEXT_PUBLIC_` prefix on API keys. They are never shipped to the browser.
3. **Pre-signed URLs for S3.** The frontend never holds AWS credentials вҖ” it gets temporary, scoped URLs from the API route.
4. **Rotation-ready.** Vercel env vars can be updated without redeploying code.

### .env.example (committed to repo)

```env
# Required
OPENAI_API_KEY=

# S3 (optional вҖ” defaults to local storage)
# AWS_ACCESS_KEY_ID=
# AWS_SECRET_ACCESS_KEY=
# S3_BUCKET=
# S3_REGION=us-east-1

# Storage mode: "s3" or "local" (default: local)
# STORAGE_MODE=local
```

## Decision Log

| Decision | Rationale |
|----------|-----------|
| Vercel over Railway/Render | Single platform for both frontend + API. Zero Docker config. Free tier sufficient. |
| S3 over Vercel Blob | Industry standard, pre-signed URL pattern, transferable to any cloud. Vercel Blob is Vercel-specific. |
| Local storage fallback | Reviewers can run the app with just an OpenAI key. No AWS account needed for evaluation. |
| No database in POC | Stateless pipeline вҖ” brief in, creatives out. No persistence needed at demo scale. |
| No Redis in POC | In-process Promise.all() is sufficient. Queue adds complexity without POC benefit. |
